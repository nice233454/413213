# Интерактивная карта туризма Беларуси

Современное веб-приложение для планирования туристических маршрутов по Республике Беларусь с интерактивной картой, базой данных достопримечательностей и системой онлайн-заказов.

## Возможности

### Для посетителей
- Интерактивная карта Беларуси с туристическими локациями
- Фильтрация по категориям достопримечательностей
- Подробная информация о каждой локации с фотографиями
- Создание индивидуального маршрута путешествия
- Оформление заявки на организацию тура

### Для администраторов
- Управление локациями на карте
- Добавление и редактирование достопримечательностей
- Просмотр и обработка заявок от клиентов
- Управление категориями локаций

## Технологии

- **Frontend**: React, TypeScript, Tailwind CSS
- **Карты**: Leaflet, OpenStreetMap
- **База данных**: Supabase (PostgreSQL)
- **Аутентификация**: Supabase Auth

## Быстрый старт

### Установка зависимостей
```bash
npm install
```

### Запуск в режиме разработки
```bash
npm run dev
```

### Сборка для продакшена
```bash
npm run build
```

## Использование

### Главная страница
1. Откройте приложение в браузере
2. Выберите категорию достопримечательностей или просмотрите все
3. Нажмите на маркер на карте для просмотра деталей локации
4. Добавьте интересующие места в маршрут
5. Нажмите "Оформить путешествие" и заполните форму

### Админ-панель
1. Перейдите по адресу `/admin`
2. Войдите используя email и пароль администратора
3. Управляйте локациями, категориями и заявками

## Создание администратора

Для создания учетной записи администратора используйте Supabase Dashboard:

1. Откройте Supabase Dashboard
2. Перейдите в раздел Authentication > Users
3. Создайте нового пользователя с email и паролем
4. Используйте эти данные для входа в админ-панель

## База данных

Приложение использует следующие таблицы:
- `categories` - категории достопримечательностей
- `locations` - туристические локации
- `orders` - заявки на путешествия
- `order_locations` - связь заявок с выбранными локациями

## Безопасность

- Row Level Security (RLS) включен для всех таблиц
- Публичный доступ только для чтения активных локаций
- Административные функции требуют аутентификации
- Безопасное хранение паролей через Supabase Auth
